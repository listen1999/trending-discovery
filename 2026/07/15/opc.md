### 金矿 2：Seedance/即梦 AI 视频提示词实战心得——反直觉结论：华丽 Prompt 大多没用

来源：@dotey（转发自 @Khazix0918 数字生命卡兹克）· 2026-07-15 01:41 · 👍117 👁8183 · engagement_rate 2.2%（本期全场最高，远超同期中位数约 0.15%-0.20%，属于"收藏率极高"的强候选）

内容类型：完整 Thread（原文一次性给出全部方法论，无需额外 web_fetch 还原）

**完整步骤（逐条列出）**
1. 反直觉前提：X 和小红书上流传的"华丽 Seedance 提示词"大多经过团队实测无效——Seedance/即梦对任何输入都会生成内容，但生成质量和 Prompt 华丽程度无关
2. Prompt 只需填四个部分：
   - 主体：先用图生图生成一张角色参考图，在即梦里新建主体后直接 @ 引用，复用率高
   - 场景：上传一张场景图即可，场景图中可带上主角，避免人物与环境比例失调
   - 音乐：只生成音效，不要生成 BGM——BGM 会干扰后期剪辑和配乐
   - 镜头：简洁写清景别和发生的事，不需要长篇描述
3. 不要往 Seedance 里放分镜故事板（花十几分钟做的带镜头运动曲线的华丽故事板，实测效果更差）
4. Prompt 不是越长越好：上传过主体/场景参考图后不需要再用文字重复描述
5. 不要给每个镜头加精确时间戳（如 [0-3秒]），模型不会真按时间戳分配时长，真正影响时长的是镜头数量，用"镜头01"这种编号方式更好
6. 不要盲目追求清晰度：目前对提示词响应最好的分辨率是 720p 而非 1080p，清晰度可后期用 Topaz 放大到 4K

**前置条件 / 适用人群**：需要用 Seedance/即梦（字节旗下）做 AI 视频内容的创作者、独立开发者做 AI 视频类产品
**国内可用性**：直接访问（即梦是字节国内产品，Seedance 是同一模型的海外版 Dreamina 底层）
**预计耗时**：读完约 5 分钟，实操验证一版 Prompt 约 10-20 分钟

**参考定价（经 web_search 核实）**：即梦（国内）约 ¥0.70/5 秒短片；Dreamina（海外版，年付促销价）约合人民币更低但续费价差距会收窄；开发者 API 按量计费约 0.10 美元/秒（720p 5 秒约 0.50 美元，约合 ¥3.4，按当前汇率 1 美元≈6.78 人民币）

**原始链接**：https://x.com/dotey/status/2077086124065874367

**深度综述**

这条信号的独特之处在于它是难得的"经过实测证伪流行做法"的内容——大部分 AI 视频教程还停留在"晒华丽 Prompt"阶段，这条心得反过来说"你在小红书上看到的千转提示词大多没用"，这种反共识判断本身就有传播力，也解释了为什么它的收藏率（2.2%）远高于本期其他任何一条内容。趋势定位上，这属于"中期验证"阶段的信号：AI 视频生成工具（Seedance/即梦、Sora、Kling 等）过去半年密集迭代，早期"怎么写复杂 Prompt"的打法正在被"用图片做主体/场景锚点、把创作精力放回故事本身"这种更结构化的工作流取代。最后一句"绝大多数技巧最终都会被模型进步吞噬，但故事不会"，值得内容创作者认真对待——它指向一个明确结论：AI 视频赛道的护城河会持续从"会不会写 Prompt"转移到"有没有好故事"，这对档位 A（内容创作者）是利好，因为叙事能力恰好是这个群体的存量资产。局限性在于：这条心得来自"海辛和阿文"（转发者匿名转述，未直接署名验证其身份与粉丝规模），核心方法论虽具体可执行，但缺乏"用了这套方法后播放量/完播率提升多少"这类结果数据支撑，读者需要自行实测验证效果。

**复制路径**
- 档位 A（内容创作者）：今天就可以按四要素（主体图、场景图、纯音效、简洁镜头描述）重做一条 Seedance/即梦视频草稿，对比之前"堆长 Prompt"的效果差异
- 档位 C（工具集成者）：如果在给客户交付 AI 视频类工作流，这套四要素结构可以直接固化成一个轻量 Prompt 模板或 Notion/飞书表单，减少客户"写不出好 Prompt"的沟通成本

### 金矿 3：BaoCut — 字幕转录 / 翻译 / 剪辑 Agent Skill（仅 Mac，免费开源）

**核心功能（聚焦对一人公司的价值）**
- 转录：本地设备端语音转文字，支持词级时间戳和 Speaker（说话人）识别
- 翻译：双语字幕生成，句子级对齐 + 按字符/秒的显示节奏控制
- 剪辑：基于文本编辑自动同步视频时间线，可直接"删除口癖、重复片段"
- 导出：支持 SRT / VTT / ASS / Markdown / 带字幕视频等多种格式
- 通过 Agent Skill 接入 Claude Code / Codex 等，可用自然语言指令直接触发，例如 `/baocut 转录并翻译视频：<视频url或路径>`

- 付费层：无（目前没有任何付费方案）

**10 分钟上手**
1. 从 baocut.app 下载安装 macOS App（要求 macOS 15+ / Apple Silicon）
2. 安装配套 Agent Skill：`npx skills add JimLiu/baocut -g -a claude-code codex -y`（一行命令，经 GitHub 仓库 README 核实）
3. 在 Claude Code / Codex 里直接对话触发 `/baocut` 指令处理视频

**与现有工具链配合**：适合和 Seedance/即梦（见金矿 2）搭配——先用即梦生成视频素材，再用 BaoCut 做字幕转录、双语翻译和口播稿的口癖清理，形成"AI 生成 + AI 剪辑"的一人视频生产线

**踩坑预警 / 已知限制（据推文原文，作者自述）**
- 仅支持 Mac（无 Windows/Linux 版本）
- 翻译速度偏慢，但作者称质量不错

**竞品对比**：经 web_search，同类开源方案有 mli/autocut（文本编辑器剪视频思路类似但无 GUI 和 Agent Skill 集成）、buxuku/SmartSub（内置 6 种 ASR 引擎 + 17+ 翻译服务商，功能更全但跨平台，非 Mac 专属体验）——BaoCut 的差异化在于"本地优先 + 原生 macOS GUI + Agent Skill 一键调用"的组合，GitHub 仓库当前约 17 星（经 web_fetch 核实），属于刚起步阶段的小众工具

**官方链接**：https://baocut.app/ ；GitHub：https://github.com/jimliu/baocut

这条信号代表的是"Agent Skill 生态"里一类越来越常见的产品形态：把一个原本需要点鼠标操作的桌面软件，包一层 CLI + Skill 说明书，让 Claude Code / Codex 这类 Agent 可以直接调用。作者宝玉本身是知名 AI 工程师和技术布道者（粉丝 23 万+），过去也持续发布类似的 baoyu-skills 系列开源项目，这次 BaoCut 延续的是同一条"内容生产力工具 + Agent 化"的产品路线。竞争格局上，字幕/剪辑类开源工具本身不稀缺（AutoCut、SmartSub 等都有一定积累），BaoCut 的护城河很薄——核心卖点"本地优先 + Mac 原生体验 + Skill 化"任何有一定工程能力的团队都可以在几周内复刻，17 星的现状也说明它还处于早期验证阶段，尚未形成规模效应。对档位 C（工具集成者）而言，真正值得学习的不是这个工具本身，而是它示范的打包思路："桌面 App 的 GUI 给人用，CLI + Skill 说明书给 Agent 用"——这个模式可以套用在任何本地工具上（PDF 处理、图片批处理、格式转换等），是一个可复制的产品化模板，而不只是一个字幕工具。风险在于：仅支持 Mac 这个限制天然砍掉了很大一部分潜在用户，如果作者没有后续 Windows 计划，这类工具的天花板会比较低。

这条信号处于"个人知识库 + Agent 化"这条趋势的中期验证阶段——过去一年 Obsidian 靠插件生态成为个人知识管理的事实标准，但"插件堆功能"的模式在 AI Agent 接入这件事上天然吃亏（每个插件都要单独适配）。OpenKnowledge 选择的路径是反过来：把 MCP、Skills、Agentic Search 做成第一等公民，笔记软件本身只是三层里最外面的一层，这个架构选择本身就值得独立开发者研究——它代表的是"AI 原生"和"AI 后加"两种产品设计哲学的分野。2.9k Star 和几乎每天都有提交的活跃度说明这不是一个玩具项目，而是有团队（Inkeep）在持续投入的正经产品，这也是它能进入本期金矿而不只是快讯的原因。对档位 C（工具集成者）而言，这类"给 Agent 提供原生读写接口的知识库"会越来越成为个人自动化工作流的底层基础设施，值得现在就上手熟悉。局限和风险：项目仍处早期，本地部署门槛（Node.js 24+、git）对档位 A（内容创作者）不够友好，短期内更适合有一定技术能力的档位 B/C 用户；另外 GPL-3.0 协议对希望基于此二次开发商业产品的开发者有一定限制，需要留意开源协议的传染性条款。
