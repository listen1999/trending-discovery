### 金矿 1：《故事转手绘视频》Skill —— 一段中文故事，自动生成会从黑白长出颜色的手绘日记动画

来源：@dotey（AI Engineer，23.2万粉丝）· 2026-07-22 13:18 · ~17h ago · 👍848 👁65,477 · 收藏 1,008（engagement_rate 1.54%，为当期中位数 0.12% 的 12 倍以上，Top 5% 强信号）
内容类型：完整 Thread + GitHub 开源项目

**完整步骤（逐条还原自原文）**
1. 输入一段中文故事文案，或上传一组按顺序排列的本地图片/漫画页
2. Skill 自动做中文分句和分镜拆解：默认一个完整句子对应一个节拍，遇到自然的动作/叙事转折才拆开长复合句（避免分镜切得太碎）
3. 为每个画面生成手写体字幕，保留原文措辞和叙事节奏
4. 底层用 Codex Image2（或 OpenAI API）生成分镜插画，画面从黑白线稿逐渐上色
5. 所有插画强制收纳在白色安全边框内，避免为了铺满屏幕裁掉人物/文字/关键细节
6. 提供两种转场：直接切换（文字→黑白线稿→彩色插画，从左向右逐渐揭示），或翻页转场（原始页面从右下角卷起，做出手绘日记翻书感）
7. 用 Remotion 完成动画渲染，输出正式版 1080×1440、预览版 720×960 的静音 H.264 视频，方便后续配音剪辑

**核心数据（经 GitHub 官方仓库页验证）**
- 仓库：`gnipbao/story-to-handdrawn-video`，390 Star，MIT 协议
- 运行环境：Node.js 20+、Python 3.10+、FFmpeg（需 `ffmpeg`/`ffprobe` 可在终端调用）、Chrome 或 Remotion 兼容浏览器
- 兼容 Agent：Codex、Claude Code、Kimi Code（原推文自述兼容性，未逐一实测）

**前置条件 / 适用人群**：需要能跑通 Node/Python/FFmpeg 环境并安装 Agent Skill，非技术背景的内容创作者较难独立完成，更适合有一定命令行基础的人自己跑，或找工具集成者代跑
**国内可用性**：GitHub 直接访问；Codex 目前国内使用依赖科学上网或第三方 API 中转（需要工具）；FFmpeg/Node/Python 均可国内直接安装
**预计耗时**：环境搭建约 30-60 分钟（视本地环境而定），单条故事渲染时间未在原文中说明

**复制路径（只写真正适用的档位）**
- 档位 A（内容创作者）：这是目前唯一一条完全免费、代码可自托管的方案，适合做童年回忆、人物故事、成长经历类内容；缺点是需要技术门槛，建议先找懂技术的朋友或档位 C 的人帮忙跑通一次流程，再自己复用
- 档位 C（工具集成者/vibe coder）：可以把"环境搭建+跑通+按客户故事定制输出"打包成一次性交付服务，卖给没有技术能力但需要这类视频内容的自媒体/知识付费从业者

**深度综述**

这条信号最有意思的地方不是效果本身，而是技术路线的选择：dotey 没有用视频生成大模型去"猜"整段动画，而是用 Codex Image2 只生成静态插画，剩下的分镜、转场、上色全部交给 Remotion 这个纯代码渲染引擎去做确定性合成。这个组合把成本压得很低——真正花钱的只有静态图生成，动画部分是程序化渲染，不存在视频模型那种"多次重跑抽卡"的隐性开销，这也是它敢完全开源、MIT 协议、clone 下来就能跑的底气所在。

从趋势定位看，这是本期"Codex 视频 Skill 开源潮"里技术含量最高、也最容易被验证的一条：GitHub 仓库真实存在，390 Star 是可查证的公开数据，不像同类推文里那些"评论区可见"的方案存在核实盲区。风险在于，它解决的是"生成动画"这个子问题，卖点集中在童年回忆/情感叙事这类垂类场景，商业化天花板取决于中文短视频市场对"黑白手绘日记体"这一美学风格的接受度和疲劳周期——一旦同类账号扎堆用同一个模板，观众很快会审美疲劳，这是复制这条路径最容易忽视的局限。

竞争格局上，同一天 dotey 还开源了另一条"拼贴动画 B-roll Skill"（口播视频配图，1,028 收藏），说明他本人正在批量占位"Codex 剪辑 Skill"这个细分工具生态，一人产出多个免费开源工具，实质是在为自己的技术影响力和潜在的付费产品（如培训/咨询）铺垫流量入口，而不是单纯做慈善。
