### 金矿 1：Vibe-coded 游戏 2 周变现 $8,500+，@dvassallo 的"小赌注"哲学再添案例

**来源：@dvassallo · 约 12h ago · 👍64 👁6,555**

**核心数据（已验证）**
- 赞助收入：$8,500（约 ¥57,800，按 1 USD ≈ 6.80 CNY）[据推文原文]
- 赞助商：n8n（工作流自动化平台），独家冠名赞助 [经 web_search 验证]
- X 创作者分成：金额未公开，据其自述为"史上最高" [未经验证]
- 玩家数：35,000 [据推文原文]
- 游戏名：FULL SEND（fullsend.game），浏览器端多人赛车游戏
- 背景：参加 @levelsio 主办的 Vibe Coding Game Jam 2026，成为第二热门游戏

**商业模式拆解**
- 这不是产品收入，是**内容变现**：游戏本身免费，收入来自（1）赞助商冠名（2）X 平台创作者广告分成
- 关键驱动力：dvassallo 本人在 X 上有大量 solopreneur 受众（~10万+关注者），连续 2 周高频发推讨论开发过程，制造了足够的话题度让赞助商主动联系
- 收入公式：社交资产（粉丝 × 参与度） × 话题持续时间 → 赞助 + 平台分成
- 成本极低：vibe coding 无需专业游戏开发能力，与儿子协作完成

**dvassallo 的更大图景**
- 2025 年初将 Small Bets 社群以 $3.6M（约 ¥2,448 万）出售给 Gumroad [经 web_search 验证，LinkedIn 公开信息]
- Small Bets 此前年利润约 $500K
- 一贯践行"产品组合 + 小赌注"策略：AWS 电子书、Twitter 课程、多个小产品并行
- fullsend.game 是该策略的最新实践——不追求产品本身赚钱，而是把开发过程变成内容

**复制路径**

**档位 A（内容创作者）：** 最直接适用。核心不是"做游戏"，而是"把一个有趣的创作过程公开记录"。在小红书/视频号上记录自己用 AI 工具做某个好玩项目的全过程（比如用 GPT Image 做绘本、用 Cursor 做小工具），持续发 2-3 周，积累足够关注度后寻求工具类品牌赞助。前提是已有一定粉丝基础（5,000+）。

**档位 C（工具集成者/vibe coder）：** 参考 dvassallo 的做法——用 Cursor/Claude Code 快速 vibe code 一个轻量级 Web 应用或游戏，重点不在产品完成度，而在过程的可看性和社交传播性。国内可考虑参加类似的 AI hackathon 或自组 build in public 挑战。

**关键约束**
- 这条收入高度依赖 dvassallo 的个人品牌和英文圈 solopreneur 受众。没有社交资产的冷启动者无法复刻同等效果
- n8n 赞助的本质是开发者营销预算，针对的是 dvassallo 的受众画像（独立开发者），不是游戏玩家
- 国内对应的赞助方可能是：飞书、Notion 中国、各类 AI 工具（如 Cursor、Monica）等面向开发者/创作者的品牌

### 金矿 2：$500 MRR 的 Polymarket 套壳 App，2 个月卖了 $4,000 —— 微型收购市场的信号

**来源：@marclou · 约 8h ago · 👍91 👁11,382**

**核心数据（已验证）**
- 成交价：$4,000（约 ¥27,200）[据推文原文]
- MRR：$500（约 ¥3,400）[据推文原文，TrustMRR 平台验证收入]
- 估值倍数：8x MRR [计算所得]
- 项目年龄：2 个月 [据推文原文]
- 交易平台：TrustMRR（Marc Lou 创建的微型收购市场）
- 产品类型：Polymarket（加密预测市场）的移动端套壳应用，使用 Superwall 做付费墙

**商业模式拆解**
- TrustMRR 的特点：通过 API 直连 Stripe/LemonSqueezy/RevenueCat/Superwall 等支付平台，**实时验证收入**，不接受截图自报 [经 web_search 验证]
- 平台已上架 842+ 个项目，累计验证收入超 $1.2B [据官网]
- 费率：卖方免费上架，成交收 3% 手续费 + 托管费
- Superwall：YC 支持的移动端付费墙 SDK，支持远程配置、A/B 测试付费方案，无需更新 App [经 web_search 验证]

**这条信号的真正价值**
不在于这个具体产品（Polymarket 套壳在国内无法直接复制），而在于：
1. **2 个月从零到被收购的速度** —— 微型产品的生命周期可以极短
2. **8x MRR 的定价锚点** —— 对于月收入 $500 级别的小产品，这是一个合理的退出倍数
3. **"建了就卖"作为一种商业模式** —— 不需要做到 $10K MRR 才能退出

**复制路径**

**档位 B（独立开发者）：** 这是最适用的档位。策略：用 Cursor/Claude Code 快速开发套壳/增强型移动 App（找一个有用户但体验差的平台，做更好的移动端），集成 Superwall 或 RevenueCat 做订阅付费，跑到 $300-500 MRR 后直接挂 TrustMRR/Acquire.com 出售。关键：选择有持续需求的垂直领域（加密、健身、生产力），而非昙花一现的热点。海外支付用 Stripe，上架用 TestFlight → App Store。

**竞争格局**
- TrustMRR 的竞品：Acquire.com（Andrew Gazdecki 创建，偏大型交易）、MicroAcquire（已改名 Acquire）、Flippa
- TrustMRR 的差异化：收入 API 验证 + 低费率，适合 $1K-50K 级别的微型交易
- 国内暂无等效平台，小型产品转让多在即刻/V2EX/独立开发者社群私下进行

### 金矿 3：Hermes Agent —— 140K Star 的开源个人 AI 助手，$5/月自建你的"参谋长"

**来源：@itsolelehmann · 约 19h ago · 👍93 👁8,286 · 📑164 bookmarks**

**核心信息（已验证）**
- 项目：Hermes Agent，由 Nous Research 开发 [经 GitHub 验证]
- GitHub：github.com/NousResearch/hermes-agent
- Star 数：~140,000 [经 web_fetch 验证]
- 最新版本：v0.13.0 [经 GitHub 验证]
- 协议：MIT（完全开源）
- 自托管成本：约 $5/月（Hetzner CX11 VPS）+ LLM API 费用（据推主自述约 $20/月）

**核心功能（聚焦一人公司价值）**
- 支持 19+ 消息平台（Telegram、Discord、WhatsApp、Slack、Signal 等），像发微信一样和 AI 助手对话
- **自学习技能系统**：每完成一个任务，自动将流程保存为可复用的 Python 技能，越用越聪明
- 持久记忆：基于 SQLite 的对话记忆，跨会话保持上下文
- 定时任务：可设置自动化流程（如每天早上扫描邮件、生成日报），服务器 7×24 运行
- 子代理：可派生多个并行任务

**定价**
- 软件：免费（MIT 开源）
- 服务器：$5/月 VPS（约 ¥34/月），无需 GPU
- LLM 费用：自带 API key，推主用 GPT-5.5 做重活 + DeepSeek 做轻活，月费约 $20（约 ¥136）
- 总计：约 ¥170/月

**10 分钟上手**
1. 准备一台 Linux VPS（Hetzner CX11 $5/月）或本地 Mac/Linux
2. 一行命令安装：`curl -sSL https://hermes-agent.org/install.sh | bash`
3. 配置 SOUL.md（个性化指令文件）和 API key
4. 连接 Telegram bot，开始对话

**国内可用性：需要工具**
- GitHub 仓库需科学上网访问
- 但支持接入国内 LLM（MiniMax、Moonshot/Kimi、GLM 等）
- 可部署在国内 VPS 上运行，消息平台可用飞书/钉钉（需自行适配）
- Telegram 在国内需科学上网

**推主实际使用场景**
- 每天早上自动扫描邮件中的客服工单，匹配 Stripe 数据，发送缺失链接、处理退款
- 据其自述"以前要花数小时的工作" [据推文原文]

**与现有工具对比**
- vs OpenClaw（@SimonHoiberg）：推主称 OpenClaw "不断出 bug 且费用高"，Hermes 更稳定且自托管成本低。但 OpenClaw 有更完整的商业化 GUI 界面
- vs Claude Code/Codex：Hermes 不是编程助手，是**生活/业务助手**，重点在消息平台集成和自动化任务

**踩坑预警**
- 安全风险：给 AI 全权限访问邮件/日历/浏览器存在 prompt injection 风险，推主 @tdinh_me 在同期推文中也提到这是最大顾虑
- 目前不建议连接生产环境数据
- Windows 支持仍在早期 beta

**复制路径**

**档位 B（独立开发者）：** 直接部署，用来自动化客服、监控竞品、处理重复性业务流程。可以先从低风险任务开始（日报生成、信息汇总），逐步扩展到客户沟通。

**档位 C（工具集成者）：** 最适用。不需要写代码就能通过自然语言配置任务流。可以把 Hermes 当成"个人 n8n"，用对话方式设置自动化。适合帮客户搭建类似系统作为服务出售。

**档位 D（服务变现者）：** 可以把"帮客户部署和配置 Hermes Agent"本身作为一项服务。140K Star 说明市场需求存在，但大部分非技术用户不会自己部署。
