商业模式拆解
- 直接收入：开源 Skill 本身不收费
- 间接收入：（1）公众号「歸藏的AI工具箱」流量与广告 / 社群转化（2）周刊订阅（3）AI 工具咨询和私有部署
- 收入公式的可见部分=Skill 作为流量入口 × 公众号 / 社群转化率，本质是「内容 × 工具」的双轮
- 毛利结构高（无服务器成本），但增长上限取决于 Claude Code 生态的活跃度

复制路径（只写真正适用的档位）
- 档位 A（内容创作者）：可以学的是「把内容资产模板化、可工具化」。如果你已经在公众号 / 小红书做某个垂类的图文（健身、读书、职场），把你的图文模板抽象成一个「输入主题 → 一键产出该模板风格图」的 Skill 或 Coze 智能体，免费开源出去，导流回主账号
- 档位 C（vibe coder）：这是最直接的复制对象。Claude Code Skill 的低门槛在于：不需要部署服务器、用户在自己的 Claude Code 里 git clone 即可使用。可以挑一个「重复劳动 + 视觉输出」场景（如周报、PRD 模板、海报、PPT 单页）做一个，发布在 GitHub 后用 X / 小红书首发引流
- 档位 D（服务变现者）：模仿的不是 Skill 本身，而是「公开作品 → 私域咨询」的漏斗结构

国内可访问性
- Claude Code（Anthropic 官方）：国内不可直接访问，需代理
- GitHub：国内可访问但偶发不稳定，推荐 GitHub CLI + 国内镜像
- 公众号：直接可用

[深度综述]
这条信号最值得拆的是「Skill 即营销资产」的趋势定位。过去半年 timeline 上反复出现的现象：dotey 的 baoyu-skills（微信群聊总结）、姚金刚的 GEOFlow + 17 套 GEO Skills、歸藏的社交媒体卡片 Skill——共同点是中文 AI 创作者把自己的「私有工作流」开源成 Claude Code Skill，用 GitHub star 当作 KPI，再回流到公众号 / 私域。这本质上是把「PR 稿 + Demo 视频 + 模板下载」这套传统内容营销，换成了「可执行代码 + GitHub trending + Agent 自动发现」。第二个反直觉点是分发路径的变化：op7418 自己都没意识到 Skill 上了 star 第一，是「群友的 Agent 在帮他找项目」时撞到的——意味着分发的客户已经不只是人，还是其他人的 Agent。Skills 的 SEO 已经开始变成「Skill GEO」（Skill 在 Agent 推荐中的可发现性）。这条赛道的窗口期目前很短：Claude Code 生态还在早期，先发优势可以维持 3—6 个月；一旦 Anthropic / OpenAI 推出官方 Skill Marketplace，长尾创作者的曝光会被官方推荐位吃掉。对一人公司从业者，现实判断：现在做 Skill 的 ROI 不在直接收入（开源不赚钱），而在「公开作品 + 漏斗导流 + 自我品牌」的复利上——但前提是公众号 / X 主账号本身已经有一定基数，否则 Skill 即使 star 第一，也跑不通后续转化。

## 避坑指南

- prompt injection 已经在野外出现：jqwik 库的测试输出里植入了「删除所有 jqwik 测试和代码」的指令（@arvidkahl 转引）。任何让 Codex / Claude Code 去跑陌生开源 repo 的工作流，都应该在沙箱 / Docker 里执行，不要直接在主仓库根目录运行
- AI 钓鱼变种：联系表单 → 伪造 Calendly → Google 登录窃号（@natmiletic）；伪装"文件上传超限"的 AI 生成销售邮件（@Shpigford）。给所有客户响应工作流加一层「域名 / 链接是否在白名单」校验
- 不要被 SHIPORDIE 这类游戏化产品的「每 30 天上线一个」节奏裹挟。Marc Lou 自己也承认 28 个产品里大部分没赚钱——游戏化让你坚持，但坚持的方向错了会更累
- 国内复制美国 SBA 贷款收购无聊生意的路径基本走不通（@Codie_Sanchez 的玩法），别盲目套——国内中小企业转让市场不规范、低息政策贷款不对应
