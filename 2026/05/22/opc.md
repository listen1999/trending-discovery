### 金矿 1：把注册流程做「长」，试用转化率翻 5 倍——一个 $40K/月 App 的反直觉打法

**类型：方法论 / 教训复盘**
来源：@thepatwalls 转发 @0xrinx（RinX）· 2026-05-21 09:39 · 👍534 👁37,242 · 收藏 741
engagement_rate：1.99%（极高，远超同期中位数 0.15%–0.20%，属 Top 5%；741 次收藏说明大量读者在存档当打法用）

**核心事实（据推文原文，收入数字未经独立验证）**
- 某 App 把注册引导从 20 个界面、3% 试用转化，改成 10–15 分钟的长流程后，试用转化率升到 15%（提升 5 倍）。
- 该 App 目前月收入 $40K（约 ¥28.4 万/月）。[据其自述，经 web_search 未找到该产品的公开收入或第三方报道佐证，金额与转化数据均属未经独立验证]
- 背后心理学：损失厌恶 / 沉没成本。用户在见到付费墙之前已经花了 10 分钟回答关于自己的问题，到付费墙时会想「都填到这儿了，不如试试」。原帖原话：「付费墙之前的每一屏，都是在为‘你为什么该付钱’积累论据。一个 App 不过就是一条销售漏斗。」

**这条打法在拆解什么**
传统产品直觉是「注册越短越好、越快让用户进来越好」。这条信号把它整个掉了个个儿：注册流程不是「障碍」，而是「销售页」。每一屏问用户一个关于他自己的问题（你的目标是什么、你最大的困扰是什么……），既是在收集个性化数据，也是在让用户对着 App 反复确认「我确实有这个需求」。等付费墙出现时，用户已经自己说服了自己。

**复制路径（只写真正适用的档位）**
- **档位 B（独立开发者）**：如果做的是订阅制 App / SaaS，本周可做一件具体的事——把现在的注册流程从「尽快进主界面」改成「先问 5–8 个关于用户处境的问题，再展示一个根据答案定制的结果页，最后才到付费墙」。RevenueCat、Superwall 这类 paywall 工具支持 A/B 测试，先拿 10% 流量跑对照，比较试用转化率，再决定是否全量。注意：这套对「需求强烈、愿意为结果付费」的工具型 App 有效，对社交/内容消费型 App 可能反而劝退。
- **档位 C（工具集成者）**：交付给客户的轻量 App / 小程序，可以把「注册即用」改成「先做一个 2 分钟的诊断问卷 + 结果报告」，诊断报告本身就是转化钩子，这套逻辑用 Typeform / 腾讯问卷 + 自动化就能拼出来。
- **档位 A（内容创作者）**：同一套逻辑可迁移到知识付费的落地页——课程详情页不要一上来就报价，先用一段「你是不是正卡在 X」的自我诊断，让读者对号入座后再看价格。

**深度综述**
这条信号的价值不在那个未经验证的 $40K 数字，而在它点破了一个被反复误读的常识。「onboarding 越短越好」是 PLG（产品驱动增长）时代被神化的教条，它默认用户的耐心是稀缺资源、要尽快让用户摸到「啊哈时刻」。但这套逻辑只在「产品自己会说话、用一次就上瘾」的场景成立——比如即时通讯、短视频。对绝大多数一人公司做的工具型产品，用户用一次根本感知不到价值，短 onboarding 等于让用户在还没产生任何投入感的时候就撞上付费墙，自然 3% 转化。把流程做长、做成「问答 + 定制化结果」，本质是用沉没成本和损失厌恶这两个行为经济学杠杆，把「要不要付钱」的决策点从「价值还没建立时」往后挪到「用户已投入 10 分钟后」。最反直觉的部分是：增加摩擦反而提高了转化——前提是这些摩擦全都用来强化「这个产品是为我定制的」这一感知，而不是单纯的表单冗余。风险也在这里：如果问卷问题和最终产品价值脱节，长流程就是纯粗暴劝退；而且 15% 的试用转化是否健康，还要看试用转付费的留存——长 onboarding 可能筛进来一批「被说服但需求不真」的用户，首月退订率会偏高。这条打法值得测，但必须连着「试用转付费率」「首月留存」一起看，单看注册转化会自我欺骗。它处在「行为设计应用于独立产品」这个趋势的中期验证阶段——海外 App 圈（尤其 Mobbin、RevenueCat 社区）已经把「问卷式 onboarding」当默认范式，国内独立 App 开发者跟进的还不多，这中间有大约半年到一年的认知差窗口。

### 金矿 2：24 小时、只用 Telegram 跟 AI 对话，造出一个能上架收钱的完整产品

**类型：方法论 / 工具实操**
来源：@tdinh_me（Tony Dinh）· 2026-05-21 06:54 · 👍79 👁15,403 · 收藏 —
engagement_rate：0.12%（绝对互动不高，但信源强：Tony Dinh 是已验证 solopreneur，bio 公开列出旗下产品月收入 $137K / $5K / $518 / $50，多年 Build in Public）

**核心事实（产品经 web_search + 官网验证）**
- 产品 Pulse（pulserevenue.app）：一个查看 Paddle 收入的原生 iOS App（Paddle 官方没有手机端）。今日 / 7 天 / 30 天环比、可搜索的交易记录、桌面小组件，数据全部留在本机、无服务器无账号。
- 商业模式：App Store 免费下载；源代码 $19（约 ¥135）一次性买断，含完整 Xcode 工程、WidgetKit 扩展、一份 AGENTS.md 架构说明，附 7 天退款保证，允许改名转售。整个 App 约 2,400 行 Swift，零第三方依赖。
- 生产方式（据其自述）：用 Claude Code 远程跑在自己的 Mac Mini 上，100% vibe coding，作者全程没读过一行代码、没打开过 Xcode。AI 用 GPT Image 2 设计、SwiftUI 开发、模拟器截图自测、推 TestFlight、提交 App Store；官网由 AI 起域名、建静态站、Lighthouse 自测、GitHub Pages 部署、配 DNS；支付接的是 Polar，AI 建店、接 API、挂到官网。作者只通过 Telegram 跟 AI 对话，24 小时跑完全程。

**复制路径（只写真正适用的档位）**
- **档位 B（独立开发者）**：可直接复刻的是「远程 AI 工作流」这套基建——把 Claude Code 跑在一台常开的机器上（Mac Mini / 云服务器），配好它对 GitHub、Cloudflare API、TestFlight 的访问权限，再挂一个 Telegram bot 当指令入口，就能做到「躺在沙发上用手机指挥 AI 改代码、自测、部署」。Claude Code、GitHub、Cloudflare 国内可直接访问；上架 iOS 需海外开发者账号（$99/年），这是真实门槛。
- **档位 C（工具集成者）**：这条信号给的是「交付方式」启发——给客户做的内部工具，可以打包成「源码 + AGENTS.md 架构说明」按固定低价（$19 这个量级）卖断，而不是按项目报价。低价买断 + 允许转售，把一次性外包变成可重复销售的数字商品。
- **档位 A（内容创作者）**：不必学技术，但「我想要一个 App，于是 24 小时做了一个」这个过程本身就是优质内容选题——把全过程录屏 / 截图做成图文，是小红书、公众号的现成爆款素材（详见传播力素材区的延伸思路）。

**踩坑预警**
- $19 卖源码不是「收入模式」，是获客和内容钩子。Tony Dinh 真正赚钱的产品在 bio 里（主力产品 $137K/月，约 ¥97 万/月），Pulse 这种小项目的作用是「证明 AI 全流程可行 + 涨粉 + 给主产品引流」。不要误以为靠卖 $19 源码能养活自己。
- 「24 小时造一个 App」能成立，关键前提是 Tony Dinh 本人是资深工程师——他知道该让 AI 做什么、能一眼看出 AI 哪里跑偏。非技术者照搬「我全程不看代码」，更可能造出一个跑不起来或带安全漏洞的东西。
- Paddle 这个产品的目标用户极窄（只有用 Paddle 收款的开发者），它是「scratch your own itch」的典型，不是市场调研出来的机会。

**深度综述**
这条信号真正的看点不是「AI 能写代码」——那已是共识——而是「一个人 + AI」第一次把产品交付的全链路（设计、编码、测试、域名、官网、部署、DNS、支付、上架）走完，且唯一的人机接口是一个 Telegram 对话框。过去一年「vibe coding」停留在「AI 帮我把功能写出来」，开发者还得自己处理上架、部署、配支付这些杂活；这条信号里这些杂活全部被 AI 接管了。把它和金矿之外的两条信号连起来看会更清楚：Cursor 5 月 18 日发布的 Composer 2.5 在多语言 SWE-Bench 上做到 79.8%、几乎追平 Opus 4.7 的 80.5%，但 token 价格只有十分之一（$0.50/百万输入、$2.50/百万输出，约 ¥3.6/¥17.8）；marclou 当天就说「Composer 2.5 治好了我的 ADHD 编码，快到我没空分心，已设为默认」。能力逼平顶配、价格砍到一折，意味着「让 AI 跑全流程」从烧钱实验变成了日常可负担的操作。这对一人公司的结构性影响是：技术实现这件事的边际成本正在趋近于零，护城河被迫从「我会做」上移到「我知道该做什么」和「我能把它分发出去」。但要泼一盆冷水——Tony Dinh 能 24 小时跑通，恰恰因为他是十年经验的工程师，他的「不看代码」是一种有判断力的放手，不是无知的放任。这正是反直觉之处：AI 把执行门槛降到地板，却把「判断力」的稀缺性抬高了。对国内读者还有一层现实障碍：iOS 上架要海外开发者账号、Polar/Paddle 这类 MoR 支付通道、海外 VPS，每一环都需要「工具」或海外身份，纯国内环境下这套流程跑不通，更适合把目标产品定在小程序 / 浏览器插件 / Web 应用这些不依赖 App Store 的形态。

来源：@vista8（向阳乔木）· 2026-05-21 11:54 · 👍771 👁53,038 · 收藏 1,145

**academic-research-skills | 把 Claude Code 变成学术研究协作者**
开源地址：github.com/Imbad0202/academic-research-skills
创建日期：2026-02-26 ｜ 最近更新：2026-05-21（活跃）
Star 数：18,093（经 GitHub API 核实；推文中「1.6w」为发推时数据，现已涨到 1.8w）
国内可用：直接访问（Claude Code、GitHub 均可直接访问；底层调用 Claude API 需要工具或海外渠道）

**核心功能（聚焦对一人公司的价值）**
- 提供一条 research → write → review → revise → finalize 的完整学术工作流，覆盖找文献、整理引用、核对数据、检查逻辑一致性这些「占时间但不烧脑」的杂活。
- 苏格拉底模式：Claude 不直接动笔，而是通过 5–15 轮提问帮你把研究问题和方法论想清楚。
- 风格校准（Style Calibration）：从你过去的作品里学你的文风。
- 官方明确声明「它不替你写论文」——定位是协作者，不是代写。
- 有一个面向 Codex CLI 的姊妹版 academic-research-skills-codex。

**安装（10 分钟上手）**
```
/plugin marketplace add Imbad0202/academic-research-skills
/plugin install academic-research-skills
```
（需 Claude Code CLI，或 v3.7.0+ 的 VS Code / JetBrains 插件）

**成本**
- 工具本身免费开源。
- 跑完一篇完整论文：约 $4–6 API 费用（约 ¥28–43）+ 2–4 小时人机协作（据项目文档）。

**复制路径（只写真正适用的档位）**
- **档位 A（内容创作者）**：这套 Skill 的「研究 → 写作 → 复审 → 修订」流程不限于学术论文，深度长文、行业报告、知识付费课程大纲同样适用。可直接拿它做「选题调研 + 初稿框架」的生产线，把人力集中在观点和案例上。
- **档位 D（服务变现者）**：做留学文书、学术润色、研究咨询的服务者，这是现成的提效工具——但要注意它的定位是「不代写、做协作」，正好契合合规的辅导/咨询服务，而非代笔。
- **档位 C（工具集成者）**：更值得研究的是「它为什么三个月冲到 1.8 万 Star」——一个聚焦单一刚需、安装两行命令、README 多语言、定位克制（明说不替你写）的 Claude Code Skill，就是一个可复制的「爆款 Skill」模板。做一个垂直场景的 Skill（如法律检索、电商文案、财务对账）开源获客，再把深度功能或服务做成付费，是当前一个明确的窗口。

**竞品对比**
同类的「学术写作 AI」多是独立 SaaS（如 Jenni AI、Paperpal），按月订阅、封闭。这个 Skill 的差异化在于：开源、跑在用户自己的 Claude Code 环境里、数据不过第三方服务器、且明确不做「一键生成全文」——靠「克制」建立了信任感。

**深度综述**
这条信号单独看是一个好用的工具，放进趋势里看是「Claude Code Skill 生态正在爆发」的一个切片。本期窗口里同时出现了好几条相关信号：宝玉（@dotey）的 ClawHub 上架了 baoyu-xhs-images、baoyu-diagram、baoyu-post-to-x 等一批个人 Skill；向阳乔木同期还发了一个 Suno 音乐生成 Skill；Josh Pigford 发布了 initialcommit 的 adversarial-code-review Skill。Skill 这个形态——一段封装好的、可被 AI 直接调用的工作流——正在变成「AI 时代的 npm 包」：分发成本极低（两行命令）、可开源获客、可挂载付费。对一人公司最实在的启发不在于「用这个 Skill 写论文」，而在于 academic-research-skills 三个月 1.8 万 Star 这条增长曲线本身——它证明了一个独立开发者 / 工具集成者，靠一个聚焦的垂直 Skill 就能在三个月内攒下一份可观的注意力资产。最值得琢磨的反直觉点是它的「克制」：在所有人都用「一键生成」当卖点的时候，它反而把「我不替你写论文、我只帮你想清楚」写进了首页——这种克制不是道德姿态，而是精准的信任设计，因为学术写作的用户最怕的就是「AI 代写被查」。风险与局限也要说清：Skill 生态目前没有成熟的变现闭环，绝大多数高 Star 的 Skill 是纯爱发电，「Star 数」和「收入」之间还没打通；而且 Skill 高度依赖 Claude Code / Codex 这些宿主平台，平台一旦改规则或自己下场做官方 Skill，第三方 Skill 的生存空间会被直接挤压。现在进场的窗口期红利真实存在，但要清醒：这是注意力红利，不是收入红利，得自己想清楚后端怎么接。

## 避坑指南

> 基于本期推文中的真实信号。

- **预填的 API Key 不是「免费午餐」，可能在花你自己的钱**：tinyfool 实测，一个开源 AI 产品的 .env 里预填了 apikey，看似作者慷慨，实际是程序从用户自己的环境变量里读出了 key——结果暴涨的账单全记在用户头上。运行任何开源 AI 项目前，务必检查 .env 和代码里对环境变量的读取逻辑，最好用临时的、限额的专用 key。
- **从 GitHub 拉的代码可能带后门**：本期的「Megalodon」供应链攻击，6 小时内向 5,561 个仓库注入了 5,718 个恶意 commit，手法是伪造 CI 机器人身份、植入带 base64 payload 的 GitHub Actions 工作流。vibe coder 习惯直接克隆陌生仓库跑起来，这正是高危行为——跑别人的 repo 前，至少检查 .github/workflows 目录和有没有可疑的编码字符串。
- **「24 小时造一个 App」是有前提的，别误读**：Tony Dinh 全程不看代码能成立，是因为他是十年经验的工程师，知道该指挥 AI 做什么、能识别它跑偏。非技术者照搬「我不看代码」，更可能产出跑不起来或带安全漏洞的东西。AI 降低的是执行门槛，没降低判断力门槛。
- **别把「卖 $19 源码」当收入模式**：那是获客和内容钩子，Tony Dinh 真正的收入在主力产品（$137K/月）。把引流动作误当成商业模式，是 Build in Public 圈常见的认知错位。
