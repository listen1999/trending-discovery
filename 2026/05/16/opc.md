### 金矿 1：Warikoo 关停 ₹100 Cr 课程业务

**来源**：@warikoo · 2026-05-15 11:49 · 👍574 👁173,745 🔖233
**engagement_rate**：0.0013（0.13%，高于同期中位数）
**信号级别**：A 级（已验证高收入 solopreneur 的实质决策）

#### 核心数据（已验证）

- 业务名：WebVeda（成立于 2020 年）
- 5 年累计：50 万付费学员、₹100 Cr 销售（约 ¥8,640 万 / $1,200 万）、₹25 Cr 利润（约 ¥2,160 万 / $300 万）
- 2024 财年单年（据 @warikoo 2024-04-12 公开数据）：₹37 Cr 收入（≈ ¥3,200 万）、390K 学员、27% EBITDA、100% 退款政策
- 关闭决定：2026-05-15 公开宣布；详细原因将于 2026-05-16 20:30（IST）YouTube 直播解释（链接：youtube.com/live/VXdxOqjEuaw）
- 多家印媒（dnaindia、newsbytesapp、liveindia）援引其本人说法，把 AI 对内容/课程业务的冲击列为主因（"makes no sense to continue"）

#### 商业模式拆解

- 定价结构：单课从几百到几千卢比不等，配 100% 退款承诺。低单价 + 高频上新（4 周左右一门新课）
- 收入公式：5 年 50 万学员 / 5 = 年均 10 万学员 × 平均客单 ₹2,000 = ₹20 Cr/年（2024 年因头部 IP 效应推到 ₹37 Cr）
- 主要成本：印度本土制作团队（10 人左右）、平台分润、营销几乎为零（全靠创始人 700K X 粉丝 + 数百万 Instagram/YouTube 自有流量）
- 利润逻辑：粉丝池足够大 → CAC 接近 0 → 27% EBITDA 的关键不在产品而在分发

#### 复制路径（只写真正适用的档位）

**档位 A（内容创作者）**：
- 关键警示——如果你的课程业务核心壁垒只是"我做了别人没做的整理"，AI 已经把这个壁垒抹平到一行 prompt
- 务实选项：一是把课程从"成品交付"转向"陪练 + 反馈"（AI 做不了的人际部分）；二是把课程包装成 SOP/Skills 卖给同行（B2B 而非 B2C）
- 不要做的事：今天再跟着 Warikoo 复刻一个"X 系列课程"路径——他自己都不干了

**档位 D（服务变现者）**：
- 真正的窗口在"私教 / 一对一咨询 / 落地辅导"——这是 AI 短期内无法替代的部分
- Warikoo 自己暗示的下一步大概率也是这个方向（直播会有更多线索，建议蹲一下复盘）

#### 竞争格局

- 国内对照：罗辑思维已在 2024 年暗示课程业务下滑；得到、樊登读书等头部知识付费平台 ARR 增长曲线已经见顶
- 印度本土对照：Cuemath、Vedantu、Unacademy 都在裁员或调整
- 一句话：这是个全球同步的趋势，不是 Warikoo 个人选择

#### 深度综述

Warikoo 这条决定的杀伤力在于**它不是一个失败者撤退，而是一个 27% EBITDA 的赢家主动退场**。过去三年中文圈"知识付费已死"喊了无数次，但反例（樊登、得到、刘润 365）一直在续命。今天这条信号是第一个"已跑通的英文 solopreneur 教育生意主动关停 + 公开把 AI 列为主因"的硬数据点。

从商业模式看，他的真正护城河从来不是"课程内容好"，而是"700K X 粉丝 + Instagram 千万级触达"——CAC 接近零的分发权。AI 把"内容生产"成本砍到趋近 0 之后，这个护城河反向变成负担：他每出一门课都是在自己粉丝资产上消耗信用，因为读者会拿任何课程内容直接喂给 ChatGPT 比较，质量不如"AI + 自己研究"就反向掉粉。这是一个非常微妙的、纯内容创业者难以察觉的死亡螺旋。

风险层面：中文一人公司绝大部分还在"做课程"这条路径上加码——尤其是"Cursor / n8n / Claude Code 实战课"这种工具型课。本期数据告诉我们的是，工具型课程的窗口周期可能不超过 3-6 个月（工具迭代速度），与其投入 2 个月做一门定型课，不如做"周更陪跑社群 + 现场答疑"。

意外的部分：Warikoo 选择直接公开关停而不是悄悄淡出。这是非常典型的高声量 IP 操作——把"关停"本身做成一次内容事件，反向积累下一阶段（大概率是高客单咨询/合伙制基金）的认知资产。中文一人公司可以借鉴这个"撤退即营销"的姿态，但前提是粉丝池足够大、信任足够深。

### 金矿 2：Hermes "Chief of Staff" Agent 的 9 个开箱工作流

**来源**：@itsolelehmann · 2026-05-15 22:15（原文）/ 03:00（自转） · 👍265 👁16,065 🔖603
**engagement_rate**：0.0375（3.75%，远超同期中位数 0.15-0.2%——同期 Top 1）
**信号级别**：B 级（具体可复刻的 Agent 工作流模板）

#### 内容类型

X 长贴 / 9 个 SKILL 模板清单。Hermes 是 Ole Lehmann 自家做的"个人助理 Agent"产品（注意他在卖自家产品，但 9 个工作流本身可拆出来在任意 Agent 框架里跑）。

#### 9 个工作流（精简还原）

1. **Daily Brief**：早 7 点拉日历 + 邮件 Top 3-5 + 天气 + 兴趣订阅 3 条头条 → Telegram 一条消息
2. **Viral Swipe File**：每晚扫 X / LinkedIn / Threads 自有发文，超过互动阈值的自动入库（hook、结构、话题、开头、数据），形成自迭代风格指纹
3. **Trending Workflows Radar**：每天扫 Reddit / X / AI 论坛，过去 24h 在涨速增长的工作流，输出 5 条内容选题
4. **Meeting Prep Briefing**：日历会议前 30 分钟，拉参会人 LinkedIn / 公司背景 + 最近邮件线程 → Telegram 一页简报
5. **Humanizer**：审计任意文本，对照 30+ AI 写作信号词（em-dash、delve、tapestry、tricolon 结构）重写为自然散文
6. **Bookmark Inbox**：监控 X 收藏夹，每条新收藏 → 三段式总结 + 自动标签 → Obsidian 入库
7. **Customer Support Cron**：早晨扫客服邮件、按问题类型分类、每周 Top 5 高频问题报告
8. **Weekly Business Report**：周一早晨拉 Stripe 收入、Newsletter 订阅、内容播放量、增粉、流失、退款 → Telegram 周对比 dashboard
9. **Obsidian Second Brain**：Discord + Telegram 全天事件每日汇总入 Obsidian，长期形成可被任意模型调用的私有知识库

#### 国内可用性

- Telegram、Obsidian、Stripe 国内需独立工具
- 可替换方案：Telegram → 飞书机器人 / 企业微信；Obsidian 国内可直接用；Stripe → 自家收款方案
- Hermes 本体是付费产品（暂未查到具体定价，但产品官网 trygotham.ai 显示按席位订阅），不推荐直接买，9 个工作流可在 Claude Code + n8n / dify / Coze 等组合复刻

#### 复制路径

**档位 C（vibe coder / 工具集成者）**：
- 把 9 条按"信息聚合 / 内容生产 / 业务监控"三类切分，先做"Daily Brief + Weekly Business Report + Bookmark Inbox"三条最容易跑起来的
- 推荐栈：n8n（自部署）+ Claude Sonnet API + 飞书 / Telegram + Obsidian + 自家飞书多维表格做记忆层
- 时间预算：单条工作流 3-6 小时跑通；全套 1-2 周

**档位 B（独立开发者）**：
- 真正的机会不在自己用 9 条，而在把这套打包卖给中小客户。北美已经有人按 $5K/月 卖"managed AI Chief of Staff"（参考 gregisenberg 同期对 Agent 业务的判断）
- 国内对应：把 Hermes 这套搬到飞书生态（飞书 CLI 已在 5 月达到 10K Star，配套程度成熟），按月度订阅或一次性交付

#### 深度综述

Hermes 这种"个人助理 Agent 产品"的核心价值不在工具本身——9 条工作流任何 vibe coder 都能复刻——而在于它把"AI agent 应该长什么样"这件事固化成了一个可视的样本。本期同步出现了 gregisenberg 17 条 AI agent 观察、飞书 CLI 1 个月 10K Star、宝玉的 Pi agent 配置、abdussalampopsy 的 Claude Code + Figma MCP，加上 Hermes 这条，**5 月中旬最强的 meta 信号是"个人 Agent 配置即生产力"**——配置一次跑一年，越早开始越占便宜。

商业模式上，Hermes 走的是 Lehmann 一贯的"教非技术用户用 AI 赚钱"路径（他主页明示）。这套打法的天花板比纯 SaaS 高——因为客户买的是"省下的时间 + 信心"，不是"软件功能"。这给国内一人公司一个具体启发：与其再做一个 Claude 包装的笔记 / 总结工具，不如做"为某个细分行业（律师 / 财税 / 跨境电商）配 SOP + Agent + 持续维护"的服务化产品。客单价从 ¥299/月（个人）到 ¥3,000/月（企业）都有空间。

风险与局限：第 5 条 "Humanizer" 是这套里最值钱、最被低估的能力——AI 生成内容反检测在 Substack / Newsletter 圈已经成刚需。但这同时也是最容易被滥用的（学生作弊、内容农场），中文产品如果做这块要先想清楚合规边界。第 9 条 Obsidian Second Brain 看起来很美，但实测用一年以上的人会发现"信息聚集 ≠ 信息可用"——没有清晰的检索目的，越大越是垃圾。

意外的部分：开源中文 ASR/对齐工具的成熟速度比所有人预期的都快。一年前大家还在比谁的 Whisper 微调 fine-tune 得好，现在阿里直接开源了 ASR + 对齐 + 1 GB 以内的轻量模型。**对中国创业者最大的红利是：模型基础设施完全国产化、可本地部署、不需要用海外 API 担忧合规**。

风险与局限：Lattifai 的开源协议、企业商用条款需要自己读一遍 README——免费 ≠ 商用免费。另外 KTV 级精度的对齐对长音频（>4h）实测仍可能掉链子，做长访谈类播客建议分段处理。

Laper.ai 这条信号的真正价值不在产品本身（剧本工具历史上从来不是大生意），**而在它把"中文创作者用 Claude Code 写自家工具卖给创作者"这条路径走完整了一遍**。10 万行代码、100% AI 写、定价分阶（free → $9 → $50）、创始人有真实编剧背景——这是非常完整的"懂行业 + 用 AI 写代码"的样本。

趋势定位：这是 2025-2026 年"vibe coding 应用层"信号的中文头部样本。可对照的对照样本是海外 Marc Lou 的 36+ SaaS 矩阵（同期他还在转 trust_mrr 上 $72 MRR Reddit 爬虫卖了 $2K 的小型并购信号）。**vibe coder 模式下"懂某个行业 × 能用 AI 写代码"的复合型创业者，比纯技术 indie hacker 的胜率高 3-5 倍**——因为前者解决的是真实痛点，后者解决的是"我能写出来什么"的伪需求。

风险与局限：剧本工具本身天花板有限——好莱坞那一套 Final Draft 不是因为好用而垄断，而是因为"行业标准"。Laper 真正能做大的方向不是"替代 Final Draft"，而是借中文短剧 / 微短剧这波内容生产爆发，做"AI 故事生产平台"——分镜、配音、配乐、自动剪辑都集成进去。这个路径可能就是赵纯想的下一步。

意外的部分：他承认"100% Claude Code 写"——这件事三个月前还会被怀疑（"100K 行 vibe code 必出问题"），今天正常人都觉得"OK 那是 Claude 4.7 嘛"。**模型能力涨上来后，"代码能不能跑"已经不是问题，"产品能不能赚钱"才是问题**。这条评估标准的转移本身就是信号。
