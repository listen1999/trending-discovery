### 金矿 1：Tibo Maker 复盘 $8M 退出 —— 「earnout 是 2 年监狱，现金也会击碎你」

**信源可信度：** A 级。Tibo 在已知高收入 solopreneur 名单内，自报构建多个产品（含 SuperX、Listfast 等），ProductHunt 年度 Maker 之一，过往 Tweet Hunter / Taplio 退出有公开报道支撑。本人 bio 与历史推文一致，bio 信号可信。

#### 核心数据（据推文原文）

- 2024 年出售 Tweet Hunter + Taplio：总价 800 万美元（约 ¥5,744 万），首付 200 万美元（约 ¥1,436 万），剩余 600 万美元（约 ¥4,308 万）按 2 年业绩对赌
- 当前新的 SaaS 组合：本年度已突破 $1M MRR（约月入 ¥718 万）[据其自述，未单独披露各产品拆分，建议视作组合口径]
- 出售形态：earnout 结构（performance-based milestones），高估值代价换 2 年绑定期

#### 商业模式拆解（基于推文 + 公开背景）

- **earnout 的真实代价：** Tibo 强调每达成一个 milestone 不再是"赢"，而是"避免成为输家"——这是 KPI 化生存的心理学解构，比合同条款本身更值得反复读
- **退出后人设套牢：** "successful guy who exited" 的标签让他半年无法发新产品。这是国内独立开发者圈层很少被讨论的隐藏成本——出货心理负担会被估值反向放大
- **复出策略：** 不再做单产品深耕，而是 SaaS 组合（Build in Public 持续公开）。这是 @marclou、@tibo_maker、@dvassallo 这条流派近 12 个月的共同选择：用产品矩阵分摊单点故障，单产品 $1–10K MRR 起跳，集合做到 $100K+ MRR
- **公开数据矛盾点：** 推文当晚 Tibo 又发了一条「如果我能买回 Tweet Hunter，我应该拿它做什么？」（quote tweet，2,847 views），半玩笑半试探。买回操作可能性低，但说明 Tibo 自己情绪上仍未脱钩——这条复盘信号的真实性更强

#### 复制路径（仅对真正适用的档位）

- **档位 B（独立开发者）：** 三条具体可操作的判断标准。第一，签 earnout 前先算最坏情况：业绩未达成时的实际到手金额。Tibo 案例如果完全未达成，800 万会缩到 200 万首付，相当于按 25% 折扣卖。第二，把"被新 owner 砍掉关键功能"提前写进合同（保留功能否决权或品牌使用权），中国独立开发者对接海外收购方时尤其要警惕。第三，把"我是否做好脱离这个产品身份"作为退出前提，而不是把出售当成"自由的开始"
- **档位 C（工具集成者 / vibe coder）：** Tibo 的新打法（SaaS 组合，每个产品 vibe coding 2 周内上线）几乎就是 vibe coder 的进阶路径。可以照搬：选 3–5 个细分需求，每个用 TanStarter 这类一站式模板 + AI 加速产出 MVP，先求广，再深耕跑得最快的那个

#### 竞争格局

国内对应路径：Build in Public 在中文圈层还属于小众，更常见的是闭门做单产品到 ¥3–10 万 MRR 后投流增长。Tibo 这类"公开建造 + 多产品组合"模式在国内的复刻最大障碍是：海外 Stripe / Lemon Squeezy 收款链路稳定，但国内对接海外用户 / 海外支付要走个人独资 + WildFire / Paddle 一类的方案，不是技术门槛而是合规门槛。

#### 深度综述

把 Tibo 这条推文放回趋势里看，最值得回味的是"AI 时代退出策略的均衡正在偏移"。2022—2023 那波 SaaS 卖家热潮里，800 万美元出售 X 类工具是赢家剧本——MicroAcquire 上随手能看到几十个类似案例。但 Tibo 今天的反悔，配合 2026 年 AI coding agent（Codex / Claude Code 这类）把"重建一个 SaaS"的成本压到一个周末，意味着：旧 SaaS 的稀缺性溢价正在塌。换句话说，买方付 800 万买下的 IP + 用户 + 增长曲线，今天用 AI 工具半年就能从零复刻出 70%——这就是为什么 Tibo 看到新 owner"做出他不会做的功能"会更痛：那不是产品在变烂，是市场在告诉他，没有他这个原创者，产品就长不回原样。

商业模式角度看，Tibo 的新组合策略并不是简单的多元化分散风险，而是利用 AI 把"产品冷启动到 $1K MRR"这个最痛苦的阶段从 3-6 个月压缩到 2-4 周。把这个动作做 10 次，能拿到 $10K MRR；做 30 次，能到 $30K MRR。组合内部互相导流（共享邮件列表、Twitter audience、SaaS Founders 圈层），获客边际成本接近零。这是当下"小作坊巨头"流派最典型的现金流公式。

风险角度，复制 Tibo 的路径有两个真坑。第一是注意力分摊：30 个产品意味着每个都不会做到极致，丧失的是单产品的"复利护城河"。Marc Lou 的 ShipFast 之所以能做到 $27K/m 就是单产品深耕的极致。第二是审美疲劳：Build in Public 对外内容产出 ≥ 产品产出，对中文创作者的"耻感门槛"更高——日本和中国创业者公开收入截图远没有欧美自然。

最被读者忽略的一句话其实是 Tibo 写在最后的："don't read what the buyer is offering, read what you'd be giving up"。这是一句具体可执行的反向尽调：写出今天你产品的 8 件最让你兴奋的事，再问自己 2 年后这些是不是还在。

### 金矿 2：歸藏 op7418 的 Claude Skills 万字复盘 —— 「Agent 不是聊天框，Skill 是普通人用好 Agent 的中间层」

**来源：** @op7418 · 2026-06-12 08:38 · 👍1,089 🔁210 💬59 🔖1,711 👁162,305 · engagement_rate 1.05%（极高，Top 5% 区间）

**信源可信度：** A 级。op7418（歸藏），154K 关注，AIGC 周刊主理人，长期产出 AI 工具 / Skills 实战内容，今年密集发布过多个传播过 100K 的 Claude Skills。本期推文是他对自己几个爆款 Skills 的系统复盘，附带 X Article 链接（推文文本本身是 Article 的导读，X Article 全文需在 x.com 内打开，未在数据中提取到正文）。

#### 核心论点（综合 op7418 当日 quote tweet 中披露的 X Article 提纲）

- 为什么 Agent 不是聊天框，为什么 Agent 会**放大人的能力差距**
- Skill 是普通用户真正用好 Agent 的关键**中间层**
- 一个好 Skill 应该怎么设计、维护、分发
- Skill 生态不能只做成"仓库列表"
- 内容、产品、案例、反馈如何形成持续迭代的飞轮

#### 对一人公司的核心意味

- **Skill 是中文 AI 工具创作者最被低估的杠杆点。** Skills 本质上是 prompt + workflow + 资源的标准化打包，门槛远低于做产品但分发效率高于纯 prompt 分享。对内容创作者来说，Skill 既能转化为公众号 / 小红书内容选题，又能直接作为可下载资源做 paywall（类似 Notion 模板生意的 AI Native 版本）
- **op7418 提到的"放大能力差距"是分析核心：** Skill 让会写好 prompt 的人产出指数级提升，反过来让大批用户成为 Skill 的消费者而非生产者。这意味着 Skill 生态早期，做"标杆 Skill 案例 + 分发渠道"的人比做"Skill 仓库工具"的人更有红利

#### 复制路径

- **档位 A（内容创作者）：** 今天可以做一件事——把过去 3 个月内被收藏 / 点赞最多的工作流贴文，反向改造成一个 Skill。例如"小红书爆款选题 Skill"、"公众号文章结构 Skill"。op7418 的方法论里强调"内容 + 产品 + 案例 + 反馈飞轮"，可以照搬：发布 Skill → 用 Skill 产出内容 → 内容引流回 Skill → 用户反馈迭代
- **档位 B（独立开发者）：** 不要急着做 Skill 仓库工具——op7418 明确否定这条路。可考虑的方向：做 Skill 的部署 / 共享 / 协作基础设施（类似 GitHub Actions 之于代码），或者做"Skill 模板店"的细分垂直版（设计 / 法律 / 编程）
- **档位 C（工具集成者 / vibe coder）：** 直接落地——选 1 个你最熟悉的工作流（如"竞品落地页拆解"、"PRD 写作"），按 op7418 框架做成 Skill，挂在公众号 / 小红书做 lead magnet。两周内可验证留存

#### 国内可访问性

- Claude 主站直接访问：claude.ai 国内目前需要工具。Claude Skills 作为 Anthropic 产品形态，国内合规商用还没有清晰路径。可访问性应标注为"需要工具"
- 国内替代：百炼 / 智谱 / Kimi 也在做类似的 Agent / Skill 形态，但生态尚未起来。短期内 Claude Skills 仍是最成熟的 Skill 形态

#### 深度综述

把 op7418 这条信号放回更大的趋势框里：从 2025 年到 2026 年中，AI 工具的分发形态经历了从 "纯 prompt 卡片"（Lex Fridman 时代）→ "GPTs / 智能体应用商店"（2024 OpenAI 推出后泡沫破裂）→ "Skill 这种轻量化、可嵌入的能力包"。OpenAI 的 GPTs 在 2024 年商店推出后未达预期，核心原因之一就是"产品形态过重 + 分发激励错位"——做 GPTs 的人没有清晰的变现路径。Skills 的优势在于它绑定在 Claude / Codex 这类成熟 agent 上，不是独立产品而是能力增强，反而降低了创作者的心理门槛。

商业模式上，Skill 经济现在处于"早期信号"阶段（mid-2026），尚未出现明显的 monetize 范式。可能的演化路径有三：一是"Skill as 内容 hook"（Notion 模板店模式），二是"Skill as SaaS 入口"（Skill 触发付费产品），三是"Skill as 教学产品"（围绕 Skill 开课 / 付费社群）。op7418 自己走的是第一条 + 第三条混合路径——AIGC 周刊 + 工具箱公众号。

反直觉的点在于：op7418 明确写"Skill 生态不能只做成仓库列表"，但市场上大量动作都是做仓库（GitHub awesome-* 风格）。这其实和 2014 年 npm 仓库 vs Yeoman generator 模板的关系类似——分发优势最后会归到能产出"高质量、有口碑、有案例"的少数节点，做仓库是规模商业，做案例是杠杆商业，一人公司选后者明显更合理。

最大的风险是 Skill 协议本身的不稳定性：Anthropic 当前 Skills 是封闭生态，OpenAI 的 Code Interpreter Skills / Codex Skills 走的是另一条路。如果押注单一平台投入 3 个月做 Skill，可能 1 年后随平台改动而沉没。op7418 隐含的对冲是把 Skill 产出做成可移植内容资产（公众号文章 + 视频）。

### 金矿 3：Simon Hoiberg 「Yap-to-content」工作流 —— 走路 + 录音 + Agent + 排期工具

**来源：** @SimonHoiberg · 2026-06-12 13:34 · 👍114 🔁17 💬3 🔖144 👁9,153 · engagement_rate 1.57%（极高，Top 5%；小众但精准）

**信源可信度：** B+。Simon Hoiberg 159K 关注，FeedHive 联合创始人（社媒内容排期工具，公开运营，有公开收入数据但本日推文未给出最新数字）。本条推文是他对自己内容工作流的拆解，可执行性强。

#### 完整步骤（来自原文 + 推断）

1. 出门走长路（瑞士山区）
2. 对 Telegram 持续语音 30–40 分钟，全程"yapping"——不写稿，纯口头联想
3. 让 OpenClaw agent（推文中提到的自定义 agent；OpenClaw 在 web_search 中未找到独立产品，可能是 Simon 自建 Anthropic-stack 工具的代称，[未经验证]）对录音做结构化梳理
4. Agent 自动输出一整周内容到 FeedHive：Carousel 图文、图形、Threads
5. 回家后审稿即可发布

#### 国内可访问性

- Telegram：需要工具
- Claude / OpenClaw agent：需要工具
- 国内替代：可用微信语音 + 飞书妙记 / 通义听悟做转写 → 接通义千问 / DeepSeek / Kimi 做结构化 → 微博 / 小红书 / 视频号定时发布工具（如蝉小红、媒小三）
- FeedHive 国内替代：禾匠定时发、千牛、零信
- 整体方案在国内**可以跑通**，链路稍长但每一节点都有成熟替代

#### 复制路径

- **档位 A（内容创作者）：** 本周可验证的最小动作——挑一个周末出门走 1 小时，全程对手机语音备忘录"自言自语"想本周想发的内容选题、爆款拆解、用户洞察。回家用通义听悟转写 → 喂给 DeepSeek，让它输出 5 条小红书文案 + 3 条公众号选题。如果产出比平时坐在桌前敲键盘强，这套就值得固化为周更工作流
- **档位 D（服务变现者）：** Yap-to-content 本质是把"专家知识 → 内容产品"的链路工业化，咨询 / 培训 / 私教完全可以用同样方式把客户对话脱敏后转成公开内容矩阵（注意脱敏 + 客户授权）

#### 关键约束

- **"yapping 质量"决定一切。** 走路时如果没有真正想表达的东西，再好的 agent 也只能产出"AI 文"——这正是当前 LinkedIn / Threads 上"AI 味儿"内容泛滥的根因。Simon 能这么用是因为他本身有内容判断和受众认知，不是被 agent 替代了判断
- **平台权重风险。** 国内平台对"AI 大批量产出"的限流近 6 个月明显加强（小红书、视频号都有 AIGC 检测）。yap-to-content 跑通后，发布节奏应该控制（每天 ≤ 3 条 / 平台），并保留至少 30% 人工原创内容

#### 深度综述

这个工作流的核心创新不在"用 AI 写内容"，而在"用 AI 做结构化转换"。AI 替代的是 Simon 过去最痛苦的环节——把脑子里的想法落到稿纸的"启动摩擦"，不是替代创意本身。所有声称"AI 解决内容焦虑"的工作流，本质都是降低启动门槛，跟创意能力无关。

为什么这条信号 engagement_rate 这么高（1.57%）？因为它的目标人群（独立创作者 / 内容创业者）天然是 Twitter / X 这类平台的高收藏用户——9K 浏览里 144 收藏，意味着每 60 个看到的人就有 1 个把它存起来准备照做。这是一条典型的"小众但精准"信号。

反直觉的是 yapping 时长——30-40 分钟连续语音不间断对绝大多数人来说很反人性。Simon 隐性透露的前提是他长期做内容、对自己声音很习惯。对国内用户更友好的改造可能是 5-10 分钟 × 3-5 次的间歇 yapping，回家后让 agent 做合并整理。

风险面上，这个工作流的**长期天花板**取决于个人能持续保持的"想说的东西"——3 个月后 yap 内容会重复，agent 会越来越像 echo chamber。Simon 自己很清楚，他平行做的是"创业 + 健身 + 家庭"多重输入，输入越丰富 yap 越有料。把这条路径当作"被动收入"是误读，它是"主动消化的工业化"。

## 避坑指南

- **earnout 对赌结构里的"高估值溢价" ≠ 实际到手现金。** Tibo 案例里 600 万对赌金额在情绪上挤压 2 年。签约前先用最坏情景测算实际拿到的总价，并问自己"如果只拿首付，是否仍愿意卖"
- **"AI 工具替代写作"是错觉。** Simon Hoiberg 的 yap-to-content 工作流，AI 替代的是结构化转换不是判断本身。在没有真正想表达的内容时，agent 只会放大空洞
- **Claude Skills 生态押注单平台风险。** op7418 长文中没明说，但 Skill 协议属于平台私有，建议把产出的 Skill 反向同步成可移植内容资产（公众号文章 / 短视频 / 开源 prompt），避免平台改动让 3 个月投入清零
